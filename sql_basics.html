2. SQL Constraint Keys

a. PRIMARY KEY

PRIMARY KEY is the column that identifies each row distinctly in the table. It acts as the main identifier in the table and ensures that each row contains a different value.

It helps to prevent duplication and null entries in the table. It is also useful to maintain the consistency, accuracy and integrity of data.

Properties of SQL PRIMARY KEY:
  • Duplicate values are not allowed. If a column is assigned as the primary key, it only contains unique values.
  • The primary key will not take NULL values.
  • Only one primary key is accepted in the table.
  • If a primary key has multiple columns, it is called a composite primary key.

b. FOREIGN KEY

The FOREIGN KEY in SQL is the column in one table that references the primary key in another table. It makes a relationship between two tables with the help of a common column that has unique rows in the column.

Foreign keys are helpful in maintaining referential integrity which makes sure that data after the relationship remains consistent and invalid data is not entered into the system. 

CREATE TABLE Employees (
    Emp_ID INT PRIMARY KEY,
    Emp_Name VARCHAR(100),
    Salary DECIMAL(10, 2),
    Dept_ID INT,
    CONSTRAINT fk_dept
    FOREIGN KEY (Dept_ID)
    REFERENCES Departments(Dept_ID)
);

c. Candidate Key

A Candidate Key is a column or set of columns that can uniquely identify each row in a table. A table can have multiple candidate keys and one of them is chosen as the Primary Key. 

CREATE TABLE Student (
    RollNo INT,
    Email VARCHAR(255),
    Phone VARCHAR(20),
    Name VARCHAR(100),
    PRIMARY KEY (RollNo)
);
Here:
RollNo, Email, and Phone could all uniquely identify a student → they are candidate keys.
RollNo is chosen as the primary key.

d. Composite Key

A key that consists of two or more columns together to uniquely identify a row. A Composite Key is a key made up of two or more columns used together to uniquely identify a record. Individual columns cannot uniquely identify the row. 

CREATE TABLE CourseRegistration (
    StudentID INT,
    CourseID INT,
    RegistrationDate DATE,
    PRIMARY KEY (StudentID, CourseID)
);

Useful when no single column is unique by itself.

e. Alternative Key

An Alternate Key is a candidate key that is NOT chosen as the primary key. 

CREATE TABLE Employee (
    EmpID INT PRIMARY KEY,
    Email VARCHAR(255) UNIQUE,
    Phone VARCHAR(20) UNIQUE,
    Name VARCHAR(100)
);
Explanation:
EmpID, Email, and Phone are candidate keys.
EmpID is chosen as the primary key.
Email and Phone become alternative keys.
