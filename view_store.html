SQL Functions, Views & Stored Procedures

SQL Functions

In MySQL, a function is a routine that takes input values, performs a calculation or operation, and returns a single value.
🔧

System Defined

e.g., Max(), Now()

👤

User-Defined (UDFs)

Custom Functions

1️⃣ Built-in (System) Functions

These are predefined by MySQL and used directly in queries.

🔹 String Functions

SELECT UPPER('mysql');
SELECT LENGTH('Hello');
SELECT CONCAT('Hello', ' ', 'World');

🔹 Numeric Functions

SELECT ABS(-10);
SELECT ROUND(12.567, 2);
SELECT CEIL(4.2);

🔹 Date & Time Functions

SELECT NOW();
SELECT CURDATE();
SELECT DATEDIFF('2026-01-10', '2026-01-01');

🔹 Aggregate Functions

Work on multiple rows and return one value

SELECT COUNT(*);
SELECT SUM(salary);
SELECT AVG(marks);
SELECT MAX(age);
SELECT MIN(age);

2️⃣ User-Defined Functions (UDFs)

You can create your own function in MySQL.

Syntax: 
DELIMITER $$

CREATE FUNCTION function_name(parameter datatype)
RETURNS datatype
DETERMINISTIC
BEGIN
    -- statements
    RETURN value;
END$$

DELIMITER ;
Creating Function: 
DELIMITER $$

CREATE FUNCTION square_num(n INT)
RETURNS INT
DETERMINISTIC
BEGIN
    RETURN n * n;
END$$

DELIMITER ;
Calling Function: 
SELECT square_num(5);

SQL Views

A SQL view is a virtual table created by SELECT query. It does not store data physically but displays data stored in underlying tables. Views help simplify complex queries, enhance security, and present data in a cleaner, customized format.
What is a View?

A view is essentially a saved SQL query that behaves like a table. Think of it as a window into your data — you see a filtered or transformed version without duplicating storage.
Syntax: 
CREATE VIEW view_name AS
SELECT column1, column2.....
FROM table_name
WHERE condition;
View Types:
• Complex views (joins, aggregates, DISTINCT, GROUP BY) → read-only.
• Simple views (single table, no aggregates) → updatable/deletable.
• Use complex views for analytics and simple views for controlled updates.

🚀 Key Takeaways

1. Create Sub Table from Existing Table

CREATE TABLE CustomerSubTable AS
SELECT CustomerID, FirstName, LastName
FROM customer;

2. Create View

CREATE VIEW mid_age_customer AS
SELECT CustomerID, FirstName, LastName
FROM customer
WHERE Age >= 50;

3. Create or Update View

CREATE OR REPLACE VIEW YoungCustomers AS
SELECT FirstName, LastName, Age, Country
FROM Customer
WHERE Age < 23;

4. Delete View

DROP VIEW mid_age_customer;

5. View with JOIN

CREATE VIEW CustomerOrdersSummary AS
SELECT
    c.CustomerID,
    c.FirstName,
    c.LastName,
    c.Country,
    COUNT(o.OrderID) AS TotalOrders,
    SUM(o.Amount) AS TotalSpent
FROM Customer c
LEFT JOIN Orders o ON c.CustomerID = o.CustomerID
GROUP BY c.CustomerID, c.FirstName, c.LastName, c.Country;

Stored Procedures

A stored procedure in SQL is a saved collection of SQL statements that can be executed as a single unit, often with input/output parameters, making database operations reusable, efficient, and secure.

What is a Stored Procedure?

Definition: A stored procedure is precompiled SQL code stored in the database. It can be executed repeatedly without rewriting the SQL each time.

Purpose: Encapsulates logic such as queries, updates, inserts, or complex calculations into one callable unit.

Parameters: Can accept input parameters (values passed in), output parameters (values returned), or both.

Execution: Called by applications, users, or other procedures using a simple command like EXEC procedure_name.
1. Creating a Stored Procedure: 
CREATE PROCEDURE GetCustomerOrders
    @CustomerID INT
AS
BEGIN
    SELECT OrderID, OrderDate, TotalAmount
    FROM Orders
    WHERE CustomerID = @CustomerID;
END;
2. Executing the Procedure: 
EXEC GetCustomerOrders @CustomerID = 101;
⚠️ Drawbacks:

• Debugging can be harder compared to application code.
• Overuse may lead to complex, hard-to-maintain procedures.
• Tightly couples logic with the database, reducing portability.
In short: Stored procedures are like "functions" inside the database — powerful tools for efficiency, security, and reusability.
