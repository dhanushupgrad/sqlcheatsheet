SQL Joins Guide

JOIN means to combine something. In case of SQL, JOIN means "to combine two or more tables". SQL joins are the basic operations performed in SQL queries to combine multiple tables in a single output table.

Types of SQL Joins

  • 1. INNER JOIN → Returns rows with matching values in both tables.
  • 2. LEFT JOIN → Returns all rows from the left table, plus matches from the right.
  • 3. RIGHT JOIN → Returns all rows from the right table, plus matches from the left.
  • 4. FULL OUTER JOIN → Returns all rows when there's a match in either table.
  • 5. CROSS JOIN → Returns Cartesian product (all combinations).
  • 6. SELF JOIN → Joins a table to itself.

1. INNER JOIN

Description: Returns only matching rows from both tables. 
SELECT e.Name, d.DeptName
FROM Employees e
INNER JOIN Departments d ON e.DeptID = d.DeptID;

2. LEFT JOIN

Description: Returns all rows from Employees, with matched Departments. 
SELECT e.Name, d.DeptName
FROM Employees e
LEFT JOIN Departments d ON e.DeptID = d.DeptID;

3. RIGHT JOIN

Description: Returns all rows from Departments, with matched Employees. 
SELECT e.Name, d.DeptName
FROM Employees e
RIGHT JOIN Departments d ON e.DeptID = d.DeptID;

4. FULL OUTER JOIN

Description: Returns all rows from both tables, matched where possible. 
SELECT e.Name, d.DeptName
FROM Employees e
FULL OUTER JOIN Departments d ON e.DeptID = d.DeptID;

5. CROSS JOIN

Description: Returns Cartesian product — all combinations. 
SELECT e.Name, d.DeptName
FROM Employees e
CROSS JOIN Departments d;

6. SELF JOIN

Description: Joins a table to itself — e.g., finding employees in the same department. 
SELECT e1.Name AS Emp1, e2.Name AS Emp2
FROM Employees e1
JOIN Employees e2 ON e1.DeptID = e2.DeptID
WHERE e1.EmpID <> e2.EmpID;
