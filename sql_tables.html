SQL Table Operations Guide

Table: A table is a collection of data, organized in terms of rows and columns. In DBMS terminology, a table is known as relation and a row as tuple.
What is a Table?

A table is used to store data in a relational database. It is a collection of rows and columns. Each column in a database table represents the attributes and each row represents a set of column values in the database table. A row of a table is also known as a record.
General Table Creation Syntax: 
CREATE TABLE [Schema].
(
      [DEFAULT ] [],
      [DEFAULT ] [],
      [DEFAULT ] [],
    []
);

1. SQL CREATE TABLE

CREATE TABLE collage (
    ID INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100)
);
CREATE TABLE Employee (
    ID INT PRIMARY KEY,
    FirstName VARCHAR(255),
    LastName VARCHAR(255),
    Email VARCHAR(255),
    AddressLine VARCHAR(255),
    City VARCHAR(255),
    collage_id INT,
    FOREIGN KEY (collage_id) REFERENCES collage(ID)
);

2. Copy Table from Another

Create a new table by copying specific columns from an existing table:

CREATE TABLE Teacher AS
SELECT ID, FirstName, Email
FROM Employee;

3. SQL INSERT INTO Statement

The SQL INSERT INTO statement is the most commonly used Data Manipulation Language (DML) command whose work is to insert data in a database. It is one of the most important and basic commands in SQL for writing data into a database table. It is used to insert a single or multiple records in a table.

Syntax 1: Inserting Single Row

INSERT INTO table_name (column1, column2 ... columnN)
VALUES (value1, value2 ... valueN);

Syntax 2: Inserting Multiple Rows

INSERT INTO table_name (column1, column2 ... columnN)
VALUES 
    (value1, value2 ... valueN),
    (value1, value2 ... valueN),
    (value1, value2 ... valueN);
Example: 
INSERT INTO Employee (FirstName, LastName, Email, AddressLine, City)
VALUES 
    ('Ankul', 'Gautam', 'ankul@gmail.com', 'E-city', 'Bangalore'),
    ('Ajeet', 'Gautam', 'ajeet@gmail.com', 'Khairpara', 'Jaunpur'),
    ('Manshi', 'Yadav', 'manshi@gmail.com', 'Mohanlalgunj', 'Lucknow');

4. SQL DROP TABLE

The DROP command is used to remove database objects like tables, databases, views, indexes, or stored procedures permanently. When we remove something by using the DROP command, we cannot roll back the action. If we want to recover the deleted data, it comes only after backing up the data. The DROP command is used in DDL statements.

Syntax: 
DROP TABLE table_name;
Example: 
CREATE DATABASE hollywood_db;

5. SQL DELETE TABLE

The DELETE statement is used to delete rows from a table. If you want to remove a specific row from a table you should use WHERE condition.

Syntax: 
DELETE FROM table_name [WHERE condition];
Note: If you do not specify the WHERE condition it will remove all the rows from the table. 
DELETE FROM table_name;

There are some more terms similar to DELETE statement like DROP statement and TRUNCATE statement but they are not exactly the same - there are some differences between them.

Difference between DELETE and TRUNCATE statements

  • DELETE: Only deletes the rows from the table based on the condition defined by WHERE clause or delete all the rows from the table when condition is not specified. But it does not free the space containing by the table.
  • TRUNCATE: Used to delete all the rows from the table and free the containing space.
Syntax: 
TRUNCATE TABLE employee;

Difference between DROP and TRUNCATE statements

  • DROP: When you use the drop statement it deletes the table's rows together with the table's definition so all the relationships of that table with other tables will no longer be valid.
  • When you drop a table:
    • Table structure will be dropped
    • Relationship will be dropped
    • Integrity constraints will be dropped
    • Access privileges will also be dropped
  • TRUNCATE: When you truncate a table, the table structure remains the same, so you will not face any of the above problems.
  • In TRUNCATE, table all records will be erased but table structure remains the same. In DROP, if you want to drop a parent table then it will say 'reference table delete first'.

6. SQL RENAME TABLE

Both the RENAME TABLE and ALTER TABLE statements help in changing the name of the table.

Syntax 1: Using RENAME TABLE

RENAME TABLE old_table_name TO new_table_name;

Syntax 2: Using ALTER TABLE

ALTER TABLE old_table_name RENAME TO new_table_name;
Examples: 
RENAME TABLE teacher TO student;
ALTER TABLE student RENAME TO teacher;

7. Copy Table Data

Syntax: 
SELECT * INTO new_table_name FROM old_table_name;

8. SQL ALTER TABLE

The ALTER TABLE statement in Structured Query Language allows you to add, modify, and delete columns of an existing table. This statement also allows database users to add and remove various SQL constraints on the existing tables.

1. ADD Column

Syntax: 
ALTER TABLE table_name ADD column_name column_definition;
Example: 
ALTER TABLE teacher ADD phone VARCHAR(10);

2. DROP Column

Syntax: 
ALTER TABLE table_name DROP COLUMN column_name;
Examples: 
ALTER TABLE Cars DROP COLUMN Car_Color;
ALTER TABLE teacher DROP phone;

3. RENAME Table

Syntax: 
ALTER TABLE table_name RENAME TO new_table_name;
Example: 
RENAME TABLE teacher TO student;

4. RENAME Column

Syntax: 
ALTER TABLE table_name RENAME COLUMN old_name TO new_name;
Example: 
ALTER TABLE teacher RENAME COLUMN phone TO mobile;
