SQL Clauses Guide

In SQL, a clause is a specific part of a query that defines how data should be retrieved, filtered, grouped, or organized. Clauses are the building blocks of SQL statements like SELECT, UPDATE, DELETE, and INSERT.

SQL WHERE Clause

One of the most crucial tasks we perform is filtering data when working with databases. Not every query requires all the rows from a table when we are interested in only a few records that match a specific condition - that's where the WHERE clause comes in.

Syntax: 
SELECT column1, column2, ...
FROM table_name
WHERE condition;

SQL AND Condition

The SQL AND condition is used in SQL query to create two or more conditions to be met. It is used in SQL SELECT, INSERT, UPDATE and DELETE statements.

Important: The SQL AND condition requires that both conditions should be met. The SQL AND condition also can be used to join multiple tables in a SQL statement.
Syntax: 
SELECT columns FROM tables WHERE condition1 AND condition2;
Examples:

1. SELECT with AND:

SELECT * 
FROM emp 
WHERE Department = "IT" AND Location = "Mumbai";

2. UPDATE with AND:

UPDATE emp 
SET Location = "Delhi" 
WHERE Department = "Marketing" AND First_Name = "John";

3. DELETE with AND:

DELETE FROM emp 
WHERE Last_Name = 'Jain' AND Location = 'Bangalore';

SQL WITH CLAUSE

The SQL WITH clause is used to provide a sub-query block which can be referenced in several places within the main SQL query. It was introduced by Oracle in Oracle 9i Release 2 database.

Syntax 1:

WITH  AS (sql_subquery_statement)
SELECT column_list 
FROM  [table_name]
[WHERE ];

Syntax 2:

WITH alias_name AS (
    sql_subquery_statement
)
SELECT column_list
FROM alias_name
WHERE condition;

GROUP BY Clause

The GROUP BY clause in MySQL is used to group rows that have the same values in one or more columns and then apply aggregate functions on each group.

Syntax: 
SELECT column_name, AGGREGATE_FUNCTION(column_name)
FROM table_name
GROUP BY column_name;
Examples:

Example 1: Count employees by city

SELECT COUNT(ID), city 
FROM employee 
GROUP BY city;

Example 2: Count with WHERE condition

SELECT city, COUNT(*) AS emp_count
FROM employee
WHERE salary > 45000
GROUP BY city;

HAVING Clause

The HAVING clause is used to filter grouped results after the GROUP BY clause is applied.

Syntax: 
SELECT column_name, AGGREGATE_FUNCTION(column_name)
FROM table_name
GROUP BY column_name
HAVING condition;
Example: 
SELECT dept, COUNT(*)
FROM employee
GROUP BY dept
HAVING COUNT(*) > 5;

GROUP BY vs HAVING - Comparison

Feature GROUP BY HAVING
Purpose Groups rows Filters groups
Works on Rows → Groups Groups
Aggregate functions Required Used in condition
Query order Before HAVING After GROUP BY
Alternative — Similar to WHERE
Note: GROUP BY groups rows into summary rows, while HAVING filters those grouped results based on aggregate conditions.

SQL Clauses Summary

WHERE - Filters individual rows before grouping
AND/OR - Combines multiple conditions
WITH - Creates temporary named result sets (CTEs)
GROUP BY - Groups rows with same values
HAVING - Filters grouped results after aggregation
